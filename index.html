A Git workflow

  1. Copy some code from elsewhere (called "cloning a repo") to begin, OR start from scratch.
  2. Do work.
  3. Review your work.
  4. Tell Git you did work. (Git calls this a "commit.")
  5. Make sure your code is up-to-date ("pull").
  6. Submit ("push") your changes.
